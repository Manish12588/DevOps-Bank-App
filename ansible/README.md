# ansible

Ansible configuration for provisioning and configuring the EC2 staging server.

## Structure

```
ansible/
├── playbook.yml          # Main playbook — configures EC2 for Docker
├── ansible.cfg           # Ansible configuration (SSH settings)
├── inventory.ini         # Auto-generated by Terraform (gitignored)
└── roles/
    └── docker/
        ├── tasks/
        │   └── main.yml  # Docker installation tasks
        └── vars/
            └── main.yml  # Docker version variables
```

## What it does

The Ansible playbook runs after Terraform provisions the EC2 instance. It:

1. Updates apt packages
2. Installs Docker CE and Docker Compose plugin
3. Adds the `ubuntu` user to the `docker` group
4. Starts and enables the Docker service

## Usage

Ansible runs automatically as part of the `infra-ec2.yml` GitHub Actions workflow. To run manually:

```bash
# inventory.ini is auto-generated by Terraform
ansible-playbook -i ansible/inventory.ini ansible/playbook.yml
```

## Note

`inventory.ini` is gitignored — it contains the EC2 IP address and is generated fresh by Terraform on each provision.
