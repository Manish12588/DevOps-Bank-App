🐳 Architecture
How this Docker learning app is structured
🎓

This is a Docker Learning App. It demonstrates a classic three-tier architecture fully containerized with Docker Compose. Each tier runs in its own container and communicates over a Docker bridge network.

Tier 1
Frontend
nginx:alpine
→
Tier 2
Backend API
node:20-alpine
→
Tier 3
Database
postgres:15
🌐 Tier 1 — Frontend

Plain HTML/CSS/JS served by Nginx.
Communicates with the backend via REST API calls to http://localhost:3000.

Port: 8080

⚙️ Tier 2 — Backend API

Node.js + Express REST API.
Handles business logic: deposit, withdraw, transfer with ACID transactions.

Port: 3000

🗄️ Tier 3 — Database

PostgreSQL 15.
Stores accounts and transactions. Data persists via a Docker named volume.

Port: 5432 (internal only)

🔗 Docker Network

All containers share the bank-network bridge network.
Services call each other by container name (e.g., backend calls db:5432).

Volume: bank-pgdata
