# terraform

Infrastructure as Code for provisioning AWS resources.

## Structure

```
terraform/
├── ec2/                        # EC2 staging environment
│   ├── main.tf                 # Provider + S3 backend
│   ├── variables.tf            # Input variables
│   ├── outputs.tf              # Output values
│   ├── ec2.tf                  # EC2 instance
│   ├── security_group.tf       # Security group
│   ├── keyPair.tf              # SSH key pair
│   ├── data.tf                 # AMI lookup
│   ├── ansible.tf              # Generates Ansible inventory
│   ├── inventory.tpl           # Inventory template
│   └── devops-bank-app-key.pub # Your SSH public key
└── eks/                        # EKS production cluster
    ├── main.tf                 # Provider + S3 backend
    ├── variables.tf            # Input variables
    ├── outputs.tf              # Output values
    ├── vpc.tf                  # VPC, subnets, NAT Gateway
    ├── eks.tf                  # EKS cluster, node group, EBS CSI
    ├── iam.tf                  # IAM roles (cluster, nodes, EBS CSI)
    └── data.tf                 # Data sources
```

## Remote State (S3)

| Workspace | State file |
|-----------|-----------|
| EC2 | `s3://devops-bank-app-tfstate/ec2/terraform.tfstate` |
| EKS | `s3://devops-bank-app-tfstate/eks/terraform.tfstate` |

## One-time S3 setup

```bash
aws s3 mb s3://devops-bank-app-tfstate --region us-west-2
aws s3api put-bucket-versioning \
  --bucket devops-bank-app-tfstate \
  --versioning-configuration Status=Enabled
```

## EC2 Workspace

Provisions staging environment: VPC (10.1.0.0/16), EC2 instance, security groups, SSH key pair. Ansible inventory is auto-generated by Terraform.

## EKS Workspace

Provisions production environment: VPC (10.0.0.0/16) with public/private subnets, EKS cluster v1.30, managed node group (2x t3.medium), OIDC provider, IAM roles, EBS CSI addon.

## Manual usage

```bash
cd terraform/ec2  # or terraform/eks
terraform init
terraform plan
terraform apply -auto-approve
terraform destroy -auto-approve
```

Or use GitHub Actions workflows.

## Hardcoded values to customize for your own AWS account

| File | Value | Description |
|------|-------|-------------|
| `*/main.tf` | `region = "us-west-2"` | AWS region |
| `*/main.tf` | `bucket = "devops-bank-app-tfstate"` | S3 state bucket |
| `eks/eks.tf` | `name = "devops-bank-eks"` | EKS cluster name |
| `eks/iam.tf` | `devops-bank-*` | IAM role names |
